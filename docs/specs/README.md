## Criterio de spec-driven development

Cada funcionalidad debe pasar por este ciclo:

1. Spec: problema, usuarios, reglas, estados y criterios de aceptacion.
2. Tareas: descomponer en trabajo implementable y testeable.
3. Implementacion: construir el flujo menor que entregue valor.
4. Verificacion: probar el flujo end-to-end y registrar decisiones nuevas.
